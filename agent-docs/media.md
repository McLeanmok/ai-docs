# Media Collection and sourcing 

## When to use 
Load this skill whenever I ask you to source media content (images, video, audio, etc.)

## Role
Within the context of this skill, you are a specialist in visual communication. The media you collect must speak clearly to the content of the website. You don't have text, so the visual content must speak for itself. Don't choose vague, overused or unattractive media content. If it helps, **refer to the copy folder** to help inform your decisions

## Organization 
- create a new folder in the info directory called "media"
- create a subfolder for each page on the website.
- In subfolders for each page, create a text file called "spec sheet". This document should contain information about the media for proper citations. Grab the:
    * Title
    * creator(s)
    * date (Day, mosh, year)
    * Container/Source
    * Url
- Name the files using standard naming conventions (keob case, lowercase, dashes over underscores, etc.), and the name should be related to the context in which it's used. 

## Image sourcing  
Collect the media from the following sites:
- [Unsplash](https://unsplash.com/)
- [Pexels](https://www.pexels.com/)
- [ShutterStock](https://www.shutterstock.com/)
- [iStock](https://www.istockphoto.com/)
- [Stocksy](https://www.stocksy.com/)
- [StockSnap](https://stocksnap.io/)
- [Deposit Photos](https://depositphotos.com/)

You may find image databases [here](https://graphicmama.com/blog/free-stock-photos-websites/)

Only download images that are freely available, and are not locked behind a paywall. Do not rip photos, and don't download content that has any watermarks or branding elements.

Alongside the files, in the spec sheets, write out 1-2 all descriptions and titles.

Save photos as:
- jpg
- png (if it has a transparent background)
- svg (if the graphic is an illustration)
- webp
- avif

minimum: 5 per page

## Audio 
For audio content, please consider the following websites:
- [Pixabay Audio](https://pixabay.com/music/)
- [Bensound](http://bensound.com/)
- [Mixxit](https://mixkit.co/free-stock-music/)
- [Storyblox](https://www.storyblocks.com/audio)
- [Magnific](https://www.magnific.com/audio/music)

You can find more databases [here](https://sfxengine.com/blog/best-royalty-free-music-sites)

for the purposes of UX, try to select clips that are succinct (under 5 minutes long)

Please save audio content as:
- mp3
- wav
- flac
- ogg

## Video
When sourcing video content, use the following databases:
- [Mixxit Video](https://mixkit.co/free-stock-video/)
- [Videezy](https://www.videezy.com/)
- [Magnific Video](https://www.magnific.com/videos)
- [Coverr](https://coverr.co/)

You may find more video databases [here](https://www.foleon.com/blog/12-sites-for-free-stock-videos)

Video clips need to be short (under 10 minutes long), and should be small file sizes.

Please download videos as .mp4 files
## Iconography 

For raw SVG Iconography, you may use the following databases:
- [Huge Icons](https://hugeicons.com/)
- [Feather Icons](https://feathericons.com/)
- [Icon Buddy](https://iconbuddy.com/)
- [Iconify](https://iconify.design/)
- [Font Awesome](https://fontawesome.com/)
- [Noun Project](https://thenounproject.com/)
- [Iconoir](https://iconoir.co)
- [All SVG Icons](https://allsvgicons.com/)
- [Bootstrap Icons](https://icons.getbootstrap.com/)

Specific guidelines for icons as follows 
* strip any colour attributes, so it's editable with CSS. Also strip anything that might interfere with css styling
* Default size is 24px x 24x
* SVG files only
